# .github (Governance & Central Administration)

## 📌 概要
本リポジトリは、Standard Architecture Blueprint (SAB) プロジェクト全体のガバナンス（統治）と共通標準を一元管理するためのセントラルリポジトリである。GitHubの特殊仕様を利用し、組織内の全リポジトリに対して「規律」と「型」を自動的に供給する役割を担う。

AP試験（共通フレーム/SLCP 2013）に基づき、システム開発の「標準化」と「構成管理」を物理的に担保する。

## 🏛️ 管理資産と役割（6つの重要資産）
本リポジトリでは、以下の6ファイルを管理している。

### 1. ガバナンス・ドキュメント
- **README.md (本ファイル)**: 統治マニュアル。本リポジトリの構成と運用ルールを定義。
- **CONTRIBUTING.md**: 開発規律。Gitフロー、コミット規約、PRの手順を定義し、変更管理を標準化。
- **CODE_OF_CONDUCT.md**: 行動規範。プロフェッショナルな議論を維持するための倫理規定。

### 2. 広報・ポータル資産
- **profile/README.md**: 帝国ポータル。Organizationのトップページに表示される全体地図（4階層アーキテクチャ図）。

### 3. 共通ワークフロー・テンプレート
- **.github/ISSUE_TEMPLATE/task.md**: 要求事項の構造化。機能/非機能要件と完了条件(DoD)を標準化。
- **.github/ISSUE_TEMPLATE/bug_report.md**: 品質管理の標準化。再現手順、期待動作、実績動作の記述を強制。

## 🛡️ 運用・ガバナンスルール

1. **一元管理の原則**: 
   全リポジトリは、本リポジトリの規律を継承すること。個別設定は技術的負債とみなす。
2. **変更管理プロセス**: 
   規律やテンプレートの変更は、本リポジトリへのPRを通じた「構成管理委員会」による審議を必須とする。
3. **DRYの徹底**: 
   共通化できるワークフロー設定等は本リポジトリへ集約し、保守性を最大化する。
4. **トレーサビリティの確保**: 
   全ての作業はIssueから開始し、テンプレートに従った論理的背景を記述することで変更理由を追跡可能にする。
